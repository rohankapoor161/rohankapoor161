## My Philosophy

**Safety gates > heroic firefighting**

The best incident is the one that never happens. I build systems that prevent bad deployments, catch issues early, and make production boring (in a good way).

**Observability > monitoring**

Monitoring tells you when something broke. Observability lets you ask "why" and get answers.

**Boring deployments > exciting war stories**

If your deployment process is exciting, it's broken. Deployments should be automated, safe, and uneventful.
